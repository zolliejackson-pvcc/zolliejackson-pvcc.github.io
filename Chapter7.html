Chapter 7: Software Engineering.

Term Definition
Software Engineering The process of developing, testing, maintaining, and evaluating computer software.
Integrated Development Environment(IDE) Software application that helps programmers develop software code.
Association for Computing Machinery The world's largest educational and scientific computing society.
Purpose of ACM Help advance the art, science, engineering, and application of computing.
Benefits of ACM membership Offers resources and tools to invent the future.
Student ACM Membership The normal membership but at a reduced price.
Institue of Electrical and Electronics Engineers(IEEE) The world's largest technical professional society.
Purpose of IEEE Encourage the progress and utilization of electrotechnology and related sciences to enhance the well-being of humanity, advance the profession, and promote the overall welfare of our members.
Benefits of IEEE membership Grants new resources, valuable opportunities, and many discounts that will help advance your career.
Student IEEE membership Gives access to travel grants, scholarships, etc on top of the normal membership benefits.
Software Life Cycle: Development Phase The process of planning and managing software development.
Requirements Analysis The process of determining user expectations for a new or modified product.
Design The process of creating methods, functions, objects, and structure of the code so it meets the needs of users.
Implementation The processes and procedures to take software applications from planning and development to the production stage.
Testing The process of evaluating and verifying that a software product or application is properly working.
Modular Implementation A technique where you separate the code into independent, interchangeable modules.
Unified Modeling Language(UML) A general-purpose visual modeling language.
Use Case Generally used as a scenario where in certain situations a piece of software might be useful.
Use Case Diagram A graphical depiction of a user's possible interactions with a system.
Data Relationships Associations between tables.
One-to-One A link between the information in two tables.
One-to-Many When one row in a table can be linked with many rows in a different table but one row in the other table can only be linked with one row in the first table.
Many-to-Many When one or more items in one table can have a relationship to one or more items in another table.
Design Pattern A general, reusable solution to a commonly occuring problem within a given context in software design.
Software Quality Assurance(SQA) A means and practice of monitoring all software engineering processes, methods, and work products to ensure compliance against defined standards.
Software Testing The process of evaluating and verifying that a software product or application does what it's supposed to do.
Glass Box Testing Examines the program structure and derives test data from the program logic/code.
Black Box Testing When a system is tested with no prior knowledge of its internal workings.
Beta Testing When a sample of the intended audience tries the product out.
Alpha Testing The initial phase of validating whether a new product will preform as expected.
Documentation A way to describe a product and the process used in creating it in formal writing.
User Documentation Information created to help users understand and use a product.
System Documentation Documents that describe the system itself and its parts.
Technical Documentation Outlines how a problem will be solved by designing and building a solution for it.
Intellectual Property Law Deals with laws to protect and enforce rights of the creators and owners of inventions.
software License States that rights of the developer and user of a piece of software.
Software Disclaimer A statement that's intended to limit the legal liability and/or obligation of the person or entity that displays it.
Best Defenses Against Software Liability Claims Contributory negligence, comparative negligence, and assumption of risk.
